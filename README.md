# 💼 IT Job Platform – Proyecto de Práctica

Este proyecto ha sido desarrollado con el objetivo de reforzar y consolidar conocimientos en desarrollo frontend, abarcando tecnologías clave como **HTML, CSS, JavaScript y React.js**. A lo largo del desarrollo, se ha seguido un enfoque progresivo, evolucionando desde una implementación básica en JavaScript puro hasta una aplicación moderna basada en React con gestión de estado y routing avanzado.

La aplicación consiste en una **plataforma de búsqueda y gestión de ofertas de empleo del sector IT**, permitiendo filtrar resultados, navegar entre distintas vistas y consumir datos desde una API.

---

## 🚀 Objetivos del Proyecto

- Reforzar fundamentos de desarrollo web (HTML, CSS, JavaScript)
- Comprender la evolución hacia arquitecturas SPA (Single Page Application)
- Practicar integración progresiva de React
- Implementar buenas prácticas en organización de código y reutilización de componentes
- Gestionar estado global y navegación avanzada

---

## 🧩 Estructura del Proyecto

El proyecto está dividido en varias etapas, cada una representando un nivel de evolución tecnológica:

### 📁 01 - JavaScript

En esta primera fase se construye la base del proyecto utilizando únicamente:

- HTML
- CSS
- JavaScript puro

**Funcionalidades:**
- Página de inicio
- Vista de listado de ofertas de empleo
- Filtrado de ofertas a partir de un archivo JSON local

Esta etapa se centra en dominar la manipulación del DOM y la lógica básica de aplicaciones web.

---

### 📁 02 - React (CDN + Babel)

En esta fase se introduce React de forma progresiva mediante CDN y Babel, sin herramientas de build modernas.

**Objetivo:**
- Entender cómo funciona React "por debajo"
- Migrar la lógica existente a componentes React básicos

**Características:**
- Uso de JSX transformado en el navegador
- Componentización inicial
- Sin bundlers ni configuración avanzada

---

### 📁 03 - React (SPA Completa)

Aquí el proyecto evoluciona hacia una aplicación moderna usando React de forma estructurada.

**Mejoras clave:**
- Creación de una **Single Page Application (SPA)**
- Componentes reutilizables
- Implementación de **custom hooks**
- Consumo de datos desde una API externa
- Sincronización del estado con la URL

Esta fase representa la consolidación del conocimiento en React.

---

### 📁 04 - Routing y Estado Global

Última etapa del proyecto, enfocada en patrones más avanzados.

**Tecnologías y conceptos:**
- **React Router**
  - Navegación entre rutas
  - Uso de `useSearchParams`
  - Estilos activos con `NavLink`
- **Zustand**
  - Gestión de estado global
  - Sincronización de estado entre componentes
- Implementación de **rutas protegidas**
